# java-lotto-precourse

## 기능 구현 목록
> 기능 작동 순서대로 작성

1. 구입 금액 입력 및 검증
    - 예외 사항
        - 숫자가 아닌 경우 -> Validator
            - 잘못된 형식입니다.
2. 구입 금액 저장 및 검증
    - 예외 사항
        - 1000원 단위(1000원 이상)가 아닌 경우 -> Money
            - 구입 금액은 1000원 단위로 입력해야 합니다.
3. 구입 금액에 맞는 로또 발행
    - 서로 다른 6개 번호로 로또 발행
4. 발행한 로또 저장
    - 예외 사항
        - 번호가 중복인 경우 -> Lotto
            - 로또 번호가 중복입니다.
        - 번호가 1 이상 45 이하의 숫자가 아닌 경우 -> Lotto
            - 로또 번호는 1부터 45 사이의 숫자여야 합니다.
        - 번호의 개수가 6개가 아닌 경우 -> Lotto
            - 로또 번호는 6개여야 합니다.
5. 발행한 로또 출력
    - 개수
    - 발행 로또 번호
6. 당첨 번호 입력 및 검증
    - 예외 사항
        - 잘못된 형식인 경우 -> Validator
            - 잘못된 형식입니다.
7. 당첨 번호 저장 및 검증
    - 예외 사항
        - 번호가 중복인 경우 -> Lotto
            - 로또 번호가 중복입니다.
        - 번호가 1 이상 45 이하의 숫자가 아닌 경우 -> Lotto
            - 로또 번호는 1부터 45 사이의 숫자여야 합니다.
        - 번호의 개수가 6개가 아닌 경우 -> Lotto
            - 로또 번호는 6개여야 합니다.
8. 보너스 번호 입력 및 검증
    - 예외 사항
        - 숫자가 아닌 경우 -> Validator
            - 잘못된 형식입니다.
9. 보너스 번호 저장 및 검증
    - 예외 사항
        - 보너스 번호가 당첨번호와 중복인 경우 -> WinningLotto
            - 당첨번호와 중복입니다.
        - 번호가 1 이상 45 이하의 숫자가 아닌 경우 -> BonusNumber
            - 로또 번호는 1부터 45 사이의 숫자여야 합니다.
10. 당첨 통계 계산
11. 당첨 통계 출력
    - 등수별 정보(일치해야 하는 숫자 개수, 상금, 당첨 개수)
    - 수익률
